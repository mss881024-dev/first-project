# Prompt: Refactor

Refactor the code in the added files following these principles:

- Single responsibility: each function/class does one thing
- No duplication: extract repeated logic into shared helpers
- Clear naming: rename anything that doesn't describe its purpose
- Small functions: break down anything over ~30 lines
- Keep existing behavior: all tests must still pass after refactor

Commit each logical change separately with a descriptive message.
